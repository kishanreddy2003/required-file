Inter Service Failure Handle using Resilience4j circuit breaker
===============================================================
You have:

Property Service

Booking Service

Payment Service

And Booking Service calls Property Service before creating booking.

Now we’ll implement Service-to-Service Circuit Breaker using Resilience4j inside Booking Service.

✅ Short Interview Answer

To implement service-level circuit breaker, we configure Resilience4j in Booking Service, apply @CircuitBreaker on the method calling Property Service (via Feign/WebClient), and define a fallback method to handle failures gracefully.

This prevents cascading failure if Property Service is down.

==========================================

Configure Circuit Breaker in application.yml (Booking Service)
===============================================================
spring.cloud.openfeign.circuitbreaker.enabled=true
management.endpoints.web.exposure.include=health,metrics,circuitbreakers
management.endpoint.health.show-details=always

# ===============================
resilience4j.circuitbreaker.instances.propertyServiceCB.registerHealthIndicator=true
resilience4j.circuitbreaker.instances.propertyServiceCB.slidingWindowType=COUNT_BASED
resilience4j.circuitbreaker.instances.propertyServiceCB.slidingWindowSize=20
resilience4j.circuitbreaker.instances.propertyServiceCB.minimumNumberOfCalls=10
resilience4j.circuitbreaker.instances.propertyServiceCB.failureRateThreshold=50
resilience4j.circuitbreaker.instances.propertyServiceCB.waitDurationInOpenState=20s
resilience4j.circuitbreaker.instances.propertyServiceCB.permittedNumberOfCallsInHalfOpenState=5
resilience4j.circuitbreaker.instances.propertyServiceCB.automaticTransitionFromOpenToHalfOpenEnabled=true
resilience4j.circuitbreaker.instances.propertyServiceCB.eventConsumerBufferSize=10

Apply Circuit Breaker in Service Layer
=======================================
@Service
@RequiredArgsConstructor
public class BookingService {

    private final PropertyClient propertyClient;

    @CircuitBreaker(name = "propertyServiceCB", fallbackMethod = "propertyFallback")
    public BookingResponse createBooking(Long propertyId, BookingRequest request) {

        // Call property service
        PropertyDto property = propertyClient.getPropertyById(propertyId);

        if(property == null) {
            throw new RuntimeException("Property not available");
        }

        // continue booking logic
        return new BookingResponse("Booking Created Successfully");
    }

    // Fallback Method
    public BookingResponse propertyFallback(Long propertyId,
                                            BookingRequest request,
                                            Throwable ex) {

        return new BookingResponse(
            "Property Service is temporarily unavailable. Please try later."
        );
    }
}

If Property Service Fails:

5 minimum calls monitored

If 50% fail

Circuit goes OPEN

Further calls directly go to fallback

After 10 seconds → HALF OPEN


Fallback method must:

Have same parameters as original method

Add Throwable as last parameter

Have same return type

Otherwise Resilience4j throws NoSuchMethodException


Even Better (Industry Best Practice)
====================================
Add Retry + Timeout:

resilience4j.retry.instances.bookingServiceRetry.maxAttempts=3
resilience4j.retry.instances.bookingServiceRetry.waitDuration=2s

resilience4j.timelimiter.instances.bookingServiceCB.timeoutDuration=3s

Then:

@Retry(name = "bookingServiceRetry")
@CircuitBreaker(name = "bookingServiceCB", fallbackMethod = "bookingFallback")

Now:

Retry 3 times

If still fails → Circuit opens

Future calls fail immediately

After 10s → Half-open trial

That is REAL production-level design.



🥈 OPTION 2 (Enterprise Way)
===========================================================

👉 Remove @CircuitBreaker annotation
👉 Use Feign fallback instead

🔥 Why FallbackFactory is Better?
Feature	fallback	fallbackFactory
Gets actual exception	❌ No	✅ Yes
Can log real error	❌ No	✅ Yes
Enterprise usage	⚠ Medium	✅ High
Debugging production issues	❌ Hard	✅ Easy
✅ Step-by-Step Implementation
🔹 1️⃣ Enable CircuitBreaker for Feign
spring.cloud.openfeign.circuitbreaker.enabled=true

Without this → fallback won’t trigger.


2️⃣ Update Feign Client
--------------------
@FeignClient(name = "PROPERTY-SERVICE",
        url = "http://localhost:9091/api/v1/property",
        configuration = FeignClientConfig .class,
        fallbackFactory = PropertyServiceFallbackFactory.class
)
public interface PropertyClient {
    @GetMapping("/property-id")
    public APIResponse<PropertyDto> getPropertyById(@RequestParam long id);

    @GetMapping("/room-available-room-id")
    public APIResponse<List<RoomAvailability>> getTotalRoomsAvailable(@RequestParam long id);

    @GetMapping("/room-id")
    public APIResponse<Rooms> getRoomType(@RequestParam long id);

}

3️⃣ Create FallbackFactory Class (Enterprise Way)
----------------------------------------------
@Component
public class PropertyServiceFallbackFactory implements FallbackFactory<PropertyClient> {
    @Override
    public PropertyClient create(Throwable cause) {
        return new PropertyClient() {
            @Override
            public APIResponse<PropertyDto> getPropertyById(long id) {
                throw new ServiceUnavailableException("Property Service is down",cause);
            }

            @Override
            public APIResponse<List<RoomAvailability>> getTotalRoomsAvailable(long id) {
                throw new ServiceUnavailableException(
                        "Availability Service is down", cause);
            }

            @Override
            public APIResponse<Rooms> getRoomType(long id) {
                throw new ServiceUnavailableException(
                        "Room Service is down", cause);
            }
        };
    }
}


Create Custom Exception
-----------------------
public class ServiceUnavailableException extends RuntimeException {

    public ServiceUnavailableException(String message, Throwable cause) {
        super(message, cause);
    }
}

Handle Globally (Enterprise Pattern)
------------------------------------
@ControllerAdvice
public class GlobalExceptionHandlerClass {
    @ExceptionHandler(ServiceUnavailableException.class)
    public ResponseEntity<Map<String,Object>> handleServiceDown(
            ServiceUnavailableException ex) {

       Map<String,Object> response=new HashMap<>();
       response.put("status",503);
       response.put("message",ex.getMessage());
       response.put("errorCode","SERVICE_DOWN");
       response.put("timestamp", Instant.now());

        return ResponseEntity.status(HttpStatus.SERVICE_UNAVAILABLE)
                .body(response);
    }
}
